# D&D 模组整理

将长篇模组文件整理为便于DM使用的结构化格式。

## 工作流程

### 1. 分析原文件结构

读取原始模组，识别以下内容：

| 内容类型 | 示例 | 处理方式 |
|----------|------|----------|
| 基础规则解释 | d20机制、AC/HP说明 | **删除**（与 dm-instructions 重复） |
| 朗读文本 | >> 开头的框内文字 | **保留**在对应章节 |
| 怪物属性 | 加粗名称、属性块 | **提取**到速查表 |
| 检定DC/技能 | 难度等级要求 | **保留** |
| 背景设定 | 世界观、NPC背景 | **保留**在概述或独立文件 |
| 附录 | 物品表、地图说明 | **提取**到 `附录/` |

### 2. 识别模组结构类型

| 类型 | 特征 | 拆分策略 |
|------|------|----------|
| **线性遭遇型** | 短篇、顺序推进 | 按遭遇/章节编号拆分 |
| **章节任务型** | 长篇、多章节 | 按章节拆分，每章含多个任务 |
| **地点驱动型** | 多地点、开放探索 | 按地点/区域拆分 |
| **故事线型** | 主线+支线任务 | 按故事线拆分，提取任务链 |

### 3. 创建文件夹结构

```
campaigns/[模组名-中文名]/
├── README.md              # 模组速查（见下方）
├── 00-overview.md         # 概述（背景、主线、NPC）
├── 01-章节名.md          # 或 01-地点A.md
├── 02-章节名.md
├── ...
└── 附录/                  # 参考资料（如有）
    ├── 怪物速查.md
    ├── NPC.md
    └── ...
```

### 4. README.md 必含内容

```markdown
# [模组名]

## 基本信息
- 类型：（线性/章节/地点/故事线）
- 建议时长：
- 建议人数：
- 角色等级：

## 故事主线
一句话概述核心情节。

## 关键NPC
| NPC | 身份 | 备注 |
|-----|------|------|
| ... | ... | ... |

## 怪物速查（如有战斗）
| 怪物 | AC | HP | 攻击 |
|------|----|----|------|
| ... | ... | ... | ... |

## 模组结构
| 文件 | 内容 |
|------|------|
| 00-overview.md | 背景设定、主线任务 |
| ... | ... |
```

### 5. 拆分原则

**保留内容**：
- 所有朗读文本（> 开头）
- 检定要求（DC、技能）
- 战斗数值
- 地点/场景描述
- 任务目标

**提取内容**：
- 怪物属性 → 速查表
- NPC数据 → 独立文件
- 大型附录（如龙语表） → 附录文件夹

**灵活处理**：
- 遭遇/任务数量少的（≤5）：可保留在一个文件
- 遭遇/任务多的：每个独立文件
- 章节型模组：章为大单位，任务为小单位

**文件大小限制**：
- 单个文件建议不超过 **50KB**
- 超过50KB的文件应继续按地点/任务拆分
- 10个城镇的章节应拆分为10个独立文件
- 包含多个地点的章节应按地点拆分

### 6. 备份原文件

整理完成后，将原始合并文件重命名为 `*_备份.md` 并保留在模组文件夹中。

## 示例：不同结构的处理

### 线性短篇
```
├── README.md
├── 00-overview.md
├── 01-开场.md
├── 02-遭遇1.md
├── ...
└── 附录/参考资料.md
```

### 章节长篇
```
├── README.md
├── 00-overview.md        # 世界观、核心区域概述
├── 01-支线任务.md        # 多个支线任务
├── 02-主线推进.md
├── 03-关键地点.md
├── ...
└── 附录/                 # 背景设定、参考资料
```

## 注意事项

- 默认输出语言：简体中文
- 先分析结构再决定拆分方式，不要机械套用模板
- 长篇模组先拆章节，再按需拆分任务
- 附录内容复杂的可进一步拆分
